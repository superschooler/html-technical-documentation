HTML Fundamentals

Structure

HTML code is composed of elements, or tags which are the letters contained in brackets. Generally, there will be an opening and a closing tag, although some exceptions apply.

The primary HTML tags used to structure text are:

<html>
This indicates to the browser that anything contained between an opening and closing <html> tag is html code.
<head>
This tag contains information about the web page and will generally contain a <title> element.
<title>
The text within the opening and closing <title> tags does not show on the web page, but it does show at the top of the browser, above the URL.
<body>
Anything you want to appear on the main browser window should appear between the <body> opening and closing tags.
<h1> through <h6>
These tags are used for headings.
<p>
This tag indicates a paragraph.
<!DOCTYPE html>
<html>
<h1>This is Our Heading</h1>
<h2>This is Our Subheading</h2>
<p>This is a paragraph.</p>
</html>

This is Our Heading

This is Our Subheading

This is a paragraph

Attributes further identify the contents of an element and are comprised of two parts: a name and a value, separated by an equals sign.

<p lang= "en-us" >Paragraph in English </p>

Paragraph

Below is a list of common attributes:

alt
Specifies an alternative text for an image when the image cannot be displayed.
disabled
Specifies that an input element should be disabled.
href
Specifies the URL for a link.
id
Specifies a unique ID for an element. On a single page, no two id attributes should be given the same value.
src
Specifies the URL for an image
style
Specifies an inline CSS style for an element.
title
Specifies extra information about an element (displayed as a tool tip).

Text

You can change the way the text appears on the screen by using additional tags:

<b>
This tag can make words and letters appear bold.
<i>
This tag can make words and letters appear italic.
<sup>
This is used to superscript text.
<sub>
This is used to subscript text.
<br />
This is a line break, and will cause any text after it to appear on the next line. This is also a self-closing tag, which is why the forward slash comes inside of it.
<hr />
This is a horizontal rule, which creates a horizontal line across the page, and is also a self-closing tag.

Semantic Markup

The following tags are semantic markup which are not intended to alter the structure of the webpage and should not be used as such. They do, however, change the way the text looks but it's important to remember that's not their sole purpose. The primary purpose to use semantic markup is to provide extra information for screen readers or search engines.

<strong>
Indicates that the content is of strong importance. The contents of this tag will show in bold by default.
<em>
Indicates emphasis that subtly changes the meaning of a sentence. The contents of this tag will show in italic by default.
<blockquote>
Used for longer quotes that take up an entire paragraph, a blockquote is usually indented by the browser.
<q>
Used for shorter quotes within a paragraph, this tag should also put quotes around the text (however Internet Explorer does not).
<abbr>
Used with a title attribute, you can type only a word's abbreviation and when a user hovers over the word they will be able to see the full term. This tag is used as such:
<abbr title= "President of the United States" >POTUS <abbr>

POTUS
<cite>
Used to indicate where a citation is from when referencing another piece of work.
<dfn>
Used to indicate the defining instance of a new term the first time you use it in a document. May or may not appear in italics depending on the browser.
<address>
Used to contain contact details for the author of the page. Often displayed in italics.
<ins>
Can be used to show that content has been inserted into a document.
<del>
Can be used to show that content has been deleted from a document.
<s>
Indicates that something is no longer accurate or relevant, but should not be deleted. Should appear as a strikethrough.

Lists

<ol>
Ordered List. These are lists where each item on the list is numbered.
<li>
List Item. Each item on the list is placed between an <li> opening tag and an </li> closing tag
<ul>
Unordered List. These are lists that begin with a bullet point, rather than a number.
<dl>
Definition List. These are made up of a set of terms along with the definitions for each of those terms.
<dt>
Definition Term. Used to contain the term being defined.
<dd>
Definition. This will appear indented from the definition term.

You can put a second list inside of an <li> element to create a sub-list or nested list.

Links

Links allow you to:

  • Move from one website to another
  • Move from one page to another on the same website
  • Move from one part of a web page to another part of the same page
  • Open a new browser window
  • Start up your email program and address a new email to someone

Links are created using the <a> element. Anything between the opening and closing <a> tag will be text the borrower can click on.
Example:

<a href= "http://www.brianschooler.com" >Brian Schooler </a>

Brian Schooler

Linking to Other Sites

When linking to another site, the href will be the full URL for the site.

Linking to Other Pages on the Same Site

When you are linking to other pages within the same site, it is not necessary to specify the domain name in the URL. If all of the website's pages are in the same folder, the value of the href attribute is just the name of the file. For example: "index.html", "about.html", etc. If the website's pages are not in the same folder, you would have to specify the child or grandchild folder as well. For example: "music/listings.html" or "music/listings/rock.html".

Email Links

To create a link that opens a user's email program and addresses an email to a specified email address, you will start the href attribute with "mailto:" followed by the email address you want the email to be sent to. Example:

<a href= "mailto:schooler94@gmail.com" >Email Brian </a>

Email Brian

Target

If you want a link to open in a new window, you would use the target="_blank" attribute and value.

Linking to a Specific Part of the Same Page

Before you can link to a specific part of a page, you must identify the points in the page that the link will go to. This can be done by using the id attribute. To link to an element using the id attribute, the value of the href attribute would start with the # symbol, followed by the id attribute you wish to jump to.

Linking to a Specific Part of Another Page

To link to a specific part of a different page, you would specify the URL in the href attribute followed by the # symbol and id of the section you'd like the link to jump to.

Images

When building a website from scratch, it is a good idea to create a folder for all of the images used on the site.
To add an image to the page, you will use the <img> tag. This tag is a self-closing tag. Two attributes are required within the <img> tag:

  • src - Tells the browser where to find the image file, which is typically a URL.
  • alt - This is where you will provide a text description of the image in case it is not able to be viewed.

Although these two attributes are the only ones required, you can also use the title attribute to provide additional information when a user hovers their mouse over it. Often, an image will also contain height and width attributes to define its size. However, the size can also be specified in CSS rather than in an image attribute.
If you place an image within paragraph tags, it will appear inline with the text unless otherwise specified.
Three Rules for Creating Images:

  1. Save images in the right format - this is usually jpeg, gif, or png
  2. Save images at the right size - Save at the same height and width as it will apear on the website for faster loading and minimizing distortion
  3. Use the correct resolution - Most computer screens show websites at 72px per square inch (ppi), so saving images at a higher resolution may cause longer download times

Image Formats

  • JPEG - Should be used when there are many different colors in a picture.
  • GIF - Should be used when a picture has an area that is filled with exactly the same color (flat color). Usually seen in drawings rather than pictures.

Figure and Figure Caption

<figure>
Used in HTML5 to contain images and their caption to keep the two associated. You can have multiple images inside one figure as long as they share the same caption
<figcaption>
Allows you to add a caption to an image. This will fall within the figure element and under the img element to produce a caption directly below the image.

Tables

A table represents information in a grid format.

Basic Table Structure

<table>
This element is used to create a table. The contents of the table are written out row by row.
<tr>
Stands for Table Row, indicates the start of each row and is followed by one or more <td> tags (depending on the number of cells in that row), completed with a closing </tr> tag.
<td>
Stands for Table Data, represents each cell of the table.

Table Headings

<th>
The Table Heading element is used the same as <td> but represents the heading for either a column or a row. This element helps people using screen readers, improves SEO, and allows for better control over the appearance of tables in CSS. This element usually displays the headings in bold font and centered in the cell.

Spanning Columns and Rows

Similar to merge cells in excel, spanning columns allows entries in a table to stretch across more than one column. Spanning columns is used with the colspan attribute inside either the <th> or <td> elements with the value equal to however many columns you would like the data to span.
Spanning Rows works the same as spanning columns, but uses the rowspan attribute to allow a piece of data to expand down a certain amount of rows. This can also be applied in either the <th> or <td> elements.

Long Tables

Using the elements for Long Tables helps people with screen readers and helps you to style these sections differently.

<thead>
The headings of the table should be inside of this element.
<tbody>
The body should be contained in this element.
<tfoot>
The footer should sit within this element.

A Long Table can also be useful if the table is so long that a user needs to scroll but you'd like to keep the contents of the header still visible.

Forms

A form allows you to collect information from visitors to your website.

Form Structure

  • <form> - Form controls live inside a <form>element. This element should alwats carry the action attribute and will usually have a method and id attribute too.
    • action - This attribute is required in a form and its value will be the URL for the page on the server that will receive the information when the form is submitted.
    • method - Forms can be sent using one of two methods, get or post. If a method attribute is not used, the form data will default to the get method.
      • With the get method, the values from the form are added to the end of the URL specified in the action attribute which is ideal for short forms (like search boxes) or when you are just retrieving data from the web server (not sending information that adds to or deletes from a database).
      • With the post method, the values are sent in what are known as HTTP headers. The post method should be used if the form allows users to upload files, is very long, contains sensitive data, or adds information to, or deletes information from, a database.
    • id - This attribute is used to identify the form distinctly from other elements on the page. Learn more about the id attribute here.

Text Input

  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="text" - This type of attribute will result in a single line of text as the input.
    • name - Identifies the form control. Required for each form control.
    • maxlength - Limits the number of characters a user may enter into the text field.

Password Input

  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="password" - Creates a text box that acts just like a single-line text input, but the characters are blocked out.
    • name - The name attribute indicates the name of the password input, which is sent to the server with the password the user enters.
    • maxlength - Limits the number of characters a user may enter into the text field.

Text Area

  • <textarea> - The <textarea> tag is used to create a multi-line text input. This tag is not an empty element and therefore required a closing tag. Any text that appears between the opening and closing tags will appear in the text box when the page loads, and the user would have to delete the text or it will be sent to the server as well (JavaScript can be used to clear this text when a user clicks in the text area).

Radio Button

  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="radio" - Allows users to pick just one of a number of options.
    • name - Identifies the form control. Required for each form control.
    • value - Indicates the value that is sent to the server for the selected option. Each of the buttons should have a different value. Additionally, a radio button cannot be deselected once selected - only the option can be changed.
    • checked - Can be used with a radio button or checkbox to indicate which value (if any) should be selected when the page loads.

Checkbox

  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="checkbox" - Allows users to select (and unselect) one or more options in answer to a question.
    • name - Identifies the form control. Required for each form control.
    • value - Indicates the value that is sent to the server for the selected option. Each of the buttons should have a different value. Additionally, a radio button cannot be deselected once selected - only the option can be changed.
    • checked - Can be used with a radio button or checkbox to indicate which value (if any) should be selected when the page loads.

Drop Down List Box

  • <select> - The <select> tag is used to create a drop down list box (also known as a select box) to select one option from a drop down list. The <select> box will contain two or more <option> elements to create the drop down. This must also have a name attribute.
    • name - Identifies the form control. Required for each form control.
  • <option> - The <option> element is used to specify options that the user can select from. The words between the opening and closing tags will be shown to the user in the drop down box.
    • value - Indicates the value that is sent to the server for the selected option. Each of the buttons should have a different value. Additionally, a radio button cannot be deselected once selected - only the option can be changed.
    • selected - Used to indicate the option that should be selected by default when the page loads. If this attribute is not used, the first option will be selected by default when the page loads.

Multiple Select Box

  • <select> - The <select> tag is used to create a drop down list box (also known as a select box) to select one option from a drop down list. The <select> box will contain two or more <option> elements to create the drop down. This must also have a name attribute.
    • size - You can turn a drop down select box into a box that shows more than one option by adding a size attribute. Its value should be the number of options you want to show at once. This attribute may have some issues with certain browsers - proceed with caution.
    • multiple - Allows users to select multiple options from the list.

File Input Box

  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="file" - Creates a box that looks like a text input followed by a browse button which will allow the user to select a file from their cmputer to be uploaded from the website. When allowing a user to upload files, the method attribute must have a value of post.

Submit Button

  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="submit" - Used to send a form to the server.
    • name - Can be used with type="submit" but is not required.
    • value - Used to control the text that appears on a button.

Image Button

  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="image" - If you want to use an image for the submit button, this will allow you to do so.

Button & Hidden Controls

  • <button> - Allows users more control over how their buttons appear, and allows other elements to appear inside of the button.
  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="hidden" - Allows developers to add values to forms that users cannot see.

Labelling Form Controls

  • <label> - Each form control should have its own label element as this makes the form accessible to vision-impaired users. The label element can be used in one of two ways:
    1. It can wrap around both the text description and the form input.
    2. It can be kept separate from the form control and use the for attribute to indicate which form control it is a label for
  • for - The for attribute states which form control the label belongs to. The value of the for attribute matches that of the id attribute on the form control it is labeling.

Grouping Form Elements

  • <fieldset> - You can group related form controls together inside the <fieldset> element. This is particularly helpful for longer forms. Most browsers will show the fieldset with a line around the edge to show how they are related.
  • <legend> - The <legend> element can come directly after the opening <fieldset> tag and contains a caption which helps identify the purpose of that group of form controls.

Form Validation

    required - Attribute that requires a field to be completed before being submitted

Date Input

  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="date" - Creates a date input in the text box

Email & URL Input

  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="email" - Allows the browser to check that the user has provided information in the correct format of an email address.
    • type="url" - Allows the browser to check that the user has provided information in the format of a URL.

Search Input

  • <input> - The <input> element is used to create several different controls. The value of the type attribute determines what kind of input they will be creating.
    • type="search" - Creates a single line text box for search queries. The <input> element should have a type attribute whose value is search
    • placeholder - value should be text that will be shown in the text box until the user clicks that area.

Extra Markup

DOCTYPE
Each web page should begin with a DOCTYPE declaration to tell the browser which version of HTML the page is using
Comments in HTML
A comment in your code can be added without appearing in your browser by placing it between <!-- and -->. Comments are helpful when you come back to your code at a later date and need to understand it. Comments can also be used to block a segment of code from appearing in the browser.
ID Attribute
Every HTML element can carry an id attribute to identify that element from other elements on the page. Its value should begin with a letter or an underscore. No two elements on the same page should have the same value for their id attribute.
Class Attribute
Every HTML element can also carry a class attribute. This can help you identify several elements as being different from other elements on the page or differentiate between links that point to your own site and links that point to other sites.
Block Elements
Elements that always start on a new line in the browser window (such as <h1>, <p>, <li>, etc.)
Inline Elements
Elements that will always appear to continue on the same line as their neighboring elements (such as <a>, <b>, <img>, etc.)
Grouping Text & Elements in a Block
The <div> element allows you to group a set of elements together in one block-level box. The primary benefit to the <div> element is that using an id or class attribute will allow you to create CSS style rules to change the appearance of all the elements contained within it.
Grouping Text & Elements Inline
The <span> element acts like an inline equivalent of the <div> element.
iFrames
An <iframe> is like a little window that has been cut into your page - and in that window you can see another page. The term iframe is an abbreviation of inline frame. A common use of iframes is to embed a Google Map into the page. The content of the iframe can be any html page. The iframe will use the following attributes:
    • src - Specifies the URL of the page to show in the frame.
    • height - Specifies the height of the iframe in pixels
    • width - Specifies the width of the iframe in pixels
    • seamless - Can be applied to an iframe where scrollbars are not desired. Does not need a value, but can be given the value of seamless.
Information About Your Pages
The <meta> element lives inside the <head> element and contains information about your web page. This is not visible to users but fulfills a number of purposes such as telling search engines about the page, who created it, and whether or not it is time sensitive. The <meta> element is an empty element and does not need a closing tag. In a meta element, the name and content attributes tend to be used together, where the name attribute specifies the property you are setting, and the content attribute specifies the value you want to give this property. The value of the name attribute can be anything, but some common uses are as follows:
    • description - This contains a description of the page. The description is commonly used by search engines to understand what the page is about and should be a maximum of 155 characters.
    • keywords - Contains a list of comma-separated words that a user might search on to find the page (may no longer have an affect on how search engines index the site)
    • robots - Indicates whether a search engine should add this page to their search results or not. A value of noindex can be used if the page should not be added. A value of nofollow can be used if search engines should add this page in their search results but not any pages that it links to.
    • author - Defines the author of the web page
    • pragma - Prevents the browser from catching the page
    • expires - Can be used to indicate when the page should expire. Must be in the format of "Fri, 04 Apr 2014 23:59:59 GMT"
